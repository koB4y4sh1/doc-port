Gemini Embedding 2 をわかりやすく整理すると、
「複数メディアを同じ意味空間で扱える埋め込みモデル」です。

Gemini Embedding 2 は、テキストだけでなく、画像・動画・音声・PDF までを 1つの統一された埋め込み空間に変換できるのが最大の特徴です。 これにより、従来の「テキスト検索中心のRAG」から、 マルチモーダル検索・推薦・分類まで広げやすくなります。

Overview

まず押さえるべきポイント

  • Gemini Embedding 2 Preview は、Google の最初の統一マルチモーダル埋め込みモデルです。
  • テキスト・画像・動画・音声・PDF を同じベクトル空間に写像できるため、異種データ横断の検索がしやすくなります。
  • 用途は、RAG、意味検索、類似検索、推薦、分類が中心です。
  • Vertex AI のモデルページでは、2026-03-10 公開の Public Preview と記載されています。
重要なのは「何を生成するか」ではなく、「意味的に近いものをどれだけ正確に近づけられるか」です。
Important distinction

混同しやすい点

  • Gemini API の Embeddings ページには、現時点で gemini-embedding-001 を使った例が多く掲載されています。
  • 一方で、モデル一覧や Vertex AI では Gemini Embedding 2 Preview が案内されています。
  • そのため、実装時は 「サンプルコードのモデル名」 と 「使いたい最新モデル」 を分けて確認する必要があります。
つまり、ドキュメント全体では埋め込みAPIの概念説明と旧来の実装例が混在しています。
実際の採用時は、対象環境で Gemini Embedding 2 が利用可能か を先に確認するのが安全です。

Specifications

主要スペック

  • 入力形式: Text / Images / Audio / Video / PDF
  • 出力: Embeddings
  • 最大入力トークン: 8,192
  • 最大シーケンス長: 8,192 tokens
  • 出力次元: 最大 3,072
  • 結果サイズの調整: output_dimensionality で小さめの次元も指定可能
  • 課金形態: Standard PayGo 対応
Media handling

メディア別の扱い

  • 画像: 最大 6 枚、MIME は image/png / image/jpeg
  • PDF: 1ファイル、最大 6 ページ
  • 動画: 音声あり最大 80 秒、音声なし最大 120 秒
  • 文書OCR: ドキュメント入力から OCR 読み取りに対応
  • 動画音声: 音声トラック抽出とフレームとの連携をサポート
Use cases

代表的なユースケース

用途 何をするか Gemini Embedding 2 が効く理由
マルチモーダル検索 テキストで画像・動画・PDF を探す 異なるメディアを同じ意味空間で比較できるため
RAG 質問に対して関連文書や関連画像を候補化する 文書だけでなく視覚情報も検索対象にできるため
推薦 商品画像や説明文から類似商品を出す 見た目と説明をまとめて近さ評価に使いやすいため
分類 画像分類・動画分類・文書整理 埋め込み後の近傍探索やクラスタリングに流し込みやすいため
コンテンツ検索 動画内の関連フレームや類似動画を探す 動画とテキスト検索をつなげやすいため
Vertex AI の埋め込み概要ページでも、画像検索・推薦・動画コンテンツ検索 などが代表例として示されています。
Task optimization

埋め込みの精度を上げる考え方

Gemini API の Embeddings ドキュメントでは、用途に合わせて task type を指定すると 埋め込み品質を最適化しやすいと説明されています。

  • SEMANTIC_SIMILARITY: 意味的な近さ比較
  • CLASSIFICATION: 分類向け
  • CLUSTERING: クラスタリング向け
  • RETRIEVAL_DOCUMENT: 検索対象ドキュメント向け
  • RETRIEVAL_QUERY: 検索クエリ向け
  • CODE_RETRIEVAL_QUERY: コード検索向け
  • QUESTION_ANSWERING / FACT_VERIFICATION: QA・検証用途向け
Design guidance

実務での見方

  • 検索用途: クエリ側と文書側で task type を分ける設計が重要です。
  • 保存コスト: ベクトルDB容量が気になる場合は、次元数の縮小を検討できます。
  • マルチモーダルRAG: テキストだけのRAGよりも前処理設計が重要になります。
  • 評価: Recall@K や nDCG などの検索指標で必ず事前評価すべきです。
「最新モデルだから自動で精度が上がる」とは限りません。
実際の品質は、データ整形・chunking・メタデータ設計・評価方法に大きく依存します。
Pricing

Gemini Embedding 2 の料金感

  • Text input: $0.2 / 1M tokens
  • Image input: $0.00012 / image
  • Video input: $0.00079 / frame
  • Audio input: $0.00016 / sec
  • Output tokens: 課金なし
Comparison hint

比較の見どころ

  • 従来のテキスト埋め込みと比較すると、Gemini Embedding 2 はマルチモーダル対応が価値の中心です。
  • 単純なテキストRAGだけなら、既存の text embedding 系も比較候補になります。
  • ただし、画像やPDF、動画も同時に検索対象にしたいなら、統一モデルの運用メリットが大きいです。
料金比較は「1件あたり」ではなく、実データ量・メディア種別・再埋め込み頻度で見るべきです。
